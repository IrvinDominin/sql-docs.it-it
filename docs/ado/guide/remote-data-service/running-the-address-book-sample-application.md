# <a name="running-the-address-book-sample-application"></a>Esegue l'applicazione di esempio Address Book
> [!IMPORTANT]
>  A partire da Windows 8 e Windows Server 2012, i componenti server di servizi desktop remoto non sono più inclusi nel sistema operativo Windows (vedere Windows 8 e [Guida alla compatibilità tra Windows Server 2012](https://www.microsoft.com/en-us/download/details.aspx?id=27416) per altri dettagli). Componenti client di servizi desktop remoto verranno rimossa in una versione futura di Windows. Evitare di usare questa funzionalità in un nuovo progetto di sviluppo e prevedere interventi di modifica nelle applicazioni in cui è attualmente implementata. Le applicazioni che utilizzano servizi desktop remoto devono eseguire la migrazione a [servizio dati WCF](http://go.microsoft.com/fwlink/?LinkId=199565).  
  
 Per eseguire l'applicazione Address Book, seguire questa procedura.

### <a name="to-run-this-application"></a>Per eseguire questa applicazione  
  
1.  Assicurarsi che Microsoft SQL Server sia in esecuzione. Fare clic su **avviare**, scegliere **programmi**, scegliere **Microsoft SQL Server 7.0**, quindi fare clic su **Service Manager**. Se è presente una freccia verde in cerchio bianco, SQL Server è in esecuzione. In caso contrario (sarà un quadrato rosso nel cerchio bianco), fare clic su **Avvia/Continua**.  
  
2.  In Microsoft Internet Explorer 4.0 o versione successiva, digitare l'indirizzo seguente:  
  
     **http://** *webserver* **/RDS/AddressBook/AddrBook.asp**  
  
     dove *webserver* è il nome del server Web in cui sono installati i componenti server di servizi desktop remoto.  
  
3.  È quindi possibile provare vari scenari di applicazione di esempio Address Book, ad esempio la ricerca di una persona dipende il proprio nome di posta elettronica, l'elenco di tutti gli utenti con il titolo "Program Manager", o record modifica esistenti. Fare clic su **trovare** per riempire la griglia dei dati con tutti i nomi disponibili.
