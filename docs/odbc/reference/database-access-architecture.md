# <a name="database-access-architecture"></a>Architettura di accesso ai database
Una delle domande per lo sviluppo di ODBC è la parte dell'architettura di accesso di database devono essere standardizzati. SQL descritte nella sezione precedente di interfacce di programmazione, embedded SQL, SQL, moduli e CLI, ovvero sono solo una parte di questa architettura. Infatti, poiché ODBC è stato utilizzato principalmente per connettere le applicazioni basati su PC minicomputer e mainframe DBMS, sono disponibili anche numerosi dei componenti di rete, alcune delle quali potrebbe essere standardizzati.
